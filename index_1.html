What it checks

Every signal comes straight from GitHub's public API and the repository's own README — computed in your browser the moment the page loads.

Star anomaly detection

Compares star velocity against repo age, commit count, and contributor count, and samples star timestamps to catch suspicious bursts.

Bus factor

Measures how concentrated the commit history is. A 30k-star project carried by one person is a very different bet than it looks.

Maintenance pulse

Days since last commit, release cadence, median time-to-first-response on issues, and the open/closed issue ratio.

README red flags

Scans for affiliate-tagged links, hype phrasing, and badge-stuffing — the marketing patterns of growth-hacked repositories.

License plain-language warning

Translates the license into one practical sentence, including copyleft implications before you build on it commercially.

FAQ

How does it work?

When you open a GitHub repository, the extension queries GitHub's public REST API for the repo's metadata, contributors, releases, issues, and a sample of star timestamps, then computes a 0–100 trust score locally in your browser. Nothing is sent to our servers — there are no servers.

Does it need my GitHub account?

No. It works without any login. Power users can optionally paste a personal access token in the settings to raise GitHub's API rate limit from 60 to 5,000 requests per hour. The token is stored only in your browser's local extension storage.

Is a low score proof that a repo is fake?

No — it's a prompt to look closer. The score surfaces patterns that correlate with hype or abandonment risk, and every flag shows the raw numbers behind it so you can judge for yourself.

How do subscriptions and refunds work?

Pro subscriptions are sold through our merchant of record, which processes payment, issues invoices, and handles applicable taxes. You receive a license key by email and activate it in the extension's settings. Cancel anytime; contact us for refund requests within 14 days of purchase.

Privacy policy

Last updated: July 2026. Repo Reality Check is built to know as little about you as possible.

What the extension collects

Nothing. The extension has no analytics, no tracking, and no backend. All analysis happens locally in your browser using GitHub's public API.

What is stored

Your settings — including an optional GitHub personal access token and, for Pro users, your license key — are stored in Chrome's local extension storage on your device only. They are never transmitted to us.

Third parties

The extension makes requests to api.github.com to read public repository data; these requests are governed by GitHub's privacy policy. Pro license keys are validated against our merchant of record's licensing API; payment details are handled entirely by the merchant of record and never touch the extension.

Permissions

The extension requests access only to github.com and api.github.com, plus local storage for settings. It does not read your browsing history or any other site.
